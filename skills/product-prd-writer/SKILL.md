# Product PRD Writer

Use this skill for Stage 1 product requirements only.

## Scope

- Create or repair `PRD.md`.
- Clarify product goals, users, scenarios, requirements, assumptions, non-goals, and acceptance criteria.
- Keep requirements testable and separate facts from assumptions.
- Use `references/prd-outline.md` as the PRD structure guide.
- Use `references/change-impact-guide.md` only when a requirement change may affect interaction, design, assets, HTML, or handoff files.

## Boundaries

- Do not write interaction specs unless the user asks for flows, states, or behavior details.
- Do not build HTML prototypes.
- Do not create `Design.md`, `Asset-Spec.md`, or developer handoff files.

## Completion

Summarize the PRD changes, files touched, assumptions, and remaining questions. If the next useful step is interaction design, recommend `product-interaction-spec` without entering it automatically.
