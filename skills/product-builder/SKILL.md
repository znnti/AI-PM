# Product Builder

## Introduction

This skill builds the product foundation. It turns rough ideas and scattered notes into clear product scope, PRD structure, assumptions, and acceptance criteria.

## Scope

- Create or repair `PRD.md`.
- Clarify product goals, users, scenarios, requirements, assumptions, non-goals, and acceptance criteria.
- Keep requirements testable and separate product facts from assumptions.
- Use `references/prd-outline.md` as the structure guide.
- Use `references/change-impact-guide.md` only when requirement changes affect downstream stages.

## Boundaries

- Do not write full interaction specs unless the user asks for behavior details.
- Do not build HTML skeletons.
- Do not create `Design.md`, `Asset-Spec.md`, or developer handoff files.

## Completion

Summarize product scope, PRD changes, assumptions, and open questions. If the next useful step is interaction design, recommend:

`interaction-designer`
